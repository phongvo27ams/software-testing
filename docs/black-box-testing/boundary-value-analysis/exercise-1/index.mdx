# Black Box Testing

## Boundary Value Analysis

Use this subsection to practice tests at the edges of an allowed range.

:::note
Boundary tests are easiest when you identify the min and max before writing inputs.
:::

### Exercise 1. Boundary Value Analysis

#### Scenario

A hotel reservation system allows stays from 1 to 14 nights.

#### Expected thinking

The interesting test values sit at the edge of the range:

- just below the minimum
- exactly the minimum
- exactly the maximum
- just above the maximum

#### Reference code

```ts
const stayLengths = [0, 1, 2, 13, 14, 15];
```
